# 小样例 08：教程式文章

这是一篇更接近帮助中心或产品文档的标准 Markdown 文章，适合检查“说明型内容”的版面表现。

## 目标

通过 3 个步骤，把一份 Markdown 文本渲染成 PNG 图片，并快速检查结果是否正常。

## 步骤一：准备输入文件

先创建一个普通的 Markdown 文件，内容里最好同时包含标题、正文、列表和一小段代码。

```markdown
# 发布说明

今天修复了登录超时问题，并优化了搜索页渲染速度。
```

## 步骤二：执行渲染

运行命令：

```bash
python3 scripts/render_examples.py --pattern 'sample_08_*.md'
```

如果你希望输出更大一些，也可以加上 `--width 1800 --scale 1.2`。

## 步骤三：检查结果

重点看下面几项：

- 标题是否足够清楚
- 正文换行是否自然
- 代码块是否有足够留白
- 整页颜色关系是否稳定

## 常见问题

**为什么同一页里中英文感觉不像一家字体？**

通常是因为渲染器找到了“能覆盖字符”的字体，但不一定找到了“最像现代中文界面”的字体。

**为什么图片清晰了，但还是不够好看？**

因为清晰度解决的是分辨率问题，而“好不好看”更多取决于字体、颜色、留白和层级比例。
